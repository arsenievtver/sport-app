# Контекст проекта sport-app

> Этот файл — «память» проекта для агента Cursor. Читай его в начале новой сессии.

## Что это

Мобильное приложение для связки **тренер + спортсмен(ы)**. Цель — регулярное использование, положительный опыт, монетизация через B2B (тренеры) и B2C (premium для спортсменов).

## Научная база

Опора на **Self-Determination Theory (SDT)** — три потребности:

| Потребность | Продуктовый смысл |
|-------------|-------------------|
| Компетентность | Прогресс, графики, челленджи, feedback |
| Автономия | Выбор упражнений, замены, переносы, self-selected rewards |
| Связанность | Тренер в центре, кооперация, персональные сообщения |

Ключевые источники: Bitrián et al. (2020, 2021), Stancu et al. (2022), Chen & Pu (2014), Hassan et al. (2019), Szinay et al. (2023).

## Антипаттерны (не делать)

- Только лидерборды и соревнование
- Статичная геймификация (novelty effect)
- Фиксированные награды без выбора
- Сухие цифры без аффективного feedback
- Принудительная социальность
- Один размер для всех

## Definition of Done (для всех stories)

- UI покрывает user story для указанной роли
- Аналитика engagement-событий
- Push ≤ 1 релевантное/день по умолчанию
- Opt-in для всего социального
- Feedback = факт + оценка
- Тренер видит действия спортсмена (замены, переносы, комментарии)
